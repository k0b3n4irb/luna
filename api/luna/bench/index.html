luna bench — run every ROM in a directory headless, detect anomalies, and write a compatibility report + one markdown “issue” per bug.

Pure luna-api consumer (api-first). Panic-safe: Emulator::step* and load_rom catch unwinds and return errors, so a single crashing ROM becomes one ❌ row instead of aborting the whole run.

Constants§

AUDIO_FLOOR 🔒
Peak |sample| above which audio counts as genuinely playing (not just DC/dither). Below it the main window heard nothing audible.
AUDIO_TAIL_FRAMES 🔒
If the main window was silent, keep stepping up to this many extra frames purely to sample audio. Many titles (esp. Konami) open with a silent publisher logo / intro and only start the music well after frame 600, so a 10 s window false-flags them as “silent”. This is the audio twin of the coproc-testing rule’s “a black smoke screenshot ≠ a bug”: a silent boot window ≠ dead audio. ~30 s at 60 Hz is enough to reach their first track.
HANG_LOOP_MAX 🔒
A static/black screen with the CPU spinning over no more than this many distinct addresses is treated as a confirmed hang (vs. an alive render/forced-blank issue, which stays ⚠️ suspect).

WARMUP_FRAME 🔒
