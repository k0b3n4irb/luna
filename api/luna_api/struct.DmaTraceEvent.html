Expand description

One DMA byte landing in VRAM ($2118/$2119), captured at transfer time. The byte value is the one the DMA actually read from the source this instant — immune to a coprocessor (e.g. Super FX) overwriting its source buffer between the transfer and a later VRAM dump (the double-buffer confound).

Fields§

§src_full: u32

24-bit A-bus source address of this byte.

§vram_word: u16

PPU VRAM word address ($2116/7 VMADD) the byte targets.

§b_offset: u8

B-bus register offset: the byte targets $2100 + b_offset (e.g. 0x18/0x19 = $2118/$2119 VRAM; 0x04 = $2104 OAM).

§value: u8

The transferred byte.

§channel: u8

DMA channel (0-7) that performed this transfer — Mesen2’s DebugEventInfo::DmaChannel (read from dma->GetActiveChannel()), driving the Event Viewer’s per-channel filter.

§frame: u64

Completed-frame counter at the start of the owning DMA burst — lets a consumer bucket DMA→VRAM bytes by frame (the per-VBlank budget check).

§line: u16

PPU scanline at the start of the owning burst.

§hclock: u16

Exact horizontal master-clock (0..1363) at the transfer — Mesen2’s GetHClock (the Event Viewer plots events at (hclock, line)).

§blank: bool

true if the burst started in the vertical-blank window (line >= vblank_start).

§force_blank: bool

true if INIDISP ($2100) forced-blank (bit 7) was set at the write. A VRAM write is safe iff blank || force_blank (V-blank or forced blank); otherwise it races active display and the PPU drops it.
