Expand description

Snapshot of the emulator’s observable state. Every field maps to something the GUI / debugger / tests want to inspect.

Fields§

§rom: Option<RomInfo>

null until a ROM is loaded.

§cpu: CpuState

CPU 65C816 registers.

§ppu: PpuState

PPU registers + occupancy.

§cpu_regs: CpuRegsState

CPU-system registers ($4200-$421F).

§scheduler: SchedulerState

Scanline scheduler.

§apu: ApuState

APU / SPC700 / DSP.

§dma: DmaState

DMA / HDMA controller registers ($420B-$420C, $43xx).

§stats: Stats

Cumulative metrics.

§sa1: Option<Sa1State>

SA-1 coprocessor CPU state, if the loaded cartridge hosts one. None for non-SA-1 carts. Diagnostic for main↔SA-1 mailbox debugging — lets you see at a glance whether the SA-1 PC is stuck in a polling loop, running random ROM bytes, or halted.

§dsp1: Option<Dsp1State>

DSP-1 (NEC uPD7725) state, if the loaded cartridge hosts one. None for non-DSP carts.
