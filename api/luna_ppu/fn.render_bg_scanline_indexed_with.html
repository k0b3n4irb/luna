Same as render_bg_scanline_with but returns CGRAM indices (instead of decoded RGB) and tags each pixel with its tilemap priority bit. Transparent pixels (colour 0 in the relevant sub-palette) return None so the compositor can route them to a lower-priority layer instead of the backdrop.
