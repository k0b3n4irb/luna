Expand description

A cartridge mapper: routes CPU bus accesses to ROM, SRAM, and coprocessor regions as defined by the mapping mode.

The mapper does NOT charge memory-speed cycles — that’s the bus’s responsibility (see [crate::Bus::io_cycle]).

Required Methods§

fn kind(&self) -> MapperKind

Identify the mapping mode.

fn read(&mut self, addr: u32) -> Option<u8>

Returns Some(byte) if addr falls inside a region this mapper owns (ROM / SRAM / coprocessor MMIO), or None if it falls outside (in which case the bus will route to WRAM / PPU / etc.).

fn write(&mut self, addr: u32, value: u8) -> bool

Mirror of Mapper::read for writes. Returns true if the mapper accepted the write (so the bus knows not to fall through), false otherwise.

fn rom_size(&self) -> usize

Size of the ROM in bytes.

fn sram_size(&self) -> usize

Size of the SRAM in bytes (0 if none).

Provided Methods§

fn save_state(&self) -> Vec<u8> ⓘ

Serialize the mapper’s MUTABLE runtime state (SRAM, coproc RAM / regs / CPU, banking latches, etc.) — NOT the ROM. Default: no state.

Used by the save-state machinery: the trait object cannot itself derive Deserialize, so the live mapper is kept and only its mutable state is round-tripped through this byte blob.

fn load_state(&mut self, _data: &[u8])

Restore mutable state produced by Mapper::save_state into the live mapper, leaving ROM intact. Default: no-op.

fn sram(&self) -> &[u8] ⓘ

Raw battery-backed SRAM bytes — the contents of a .srm file. Empty if the cartridge has none. Used for cross-run persistence (a true power-cycle test), distinct from a full save-state.

fn load_sram(&mut self, _data: &[u8])

Load raw battery SRAM (e.g. from a .srm file). Copies up to the smaller of the two lengths; no-op if the cartridge has no SRAM.

fn reset(&mut self)

Re-power the cartridge coprocessor to its power-on state, as the SNES reset line does on real hardware (ares SuperFX::power() / SA1::power()). ROM and battery-backed SRAM persist; the coprocessor’s registers, internal RAM, caches and its own CPU return to power-on. Default = no-op for plain LoROM / HiROM carts, which have no coprocessor state to clear.

Without this, Snes::reset re-runs the main-CPU reset vector but leaves a coprocessor (Super FX / SA-1) mid-execution, so a reset of a Super FX title (e.g. Doom) freezes instead of rebooting.

fn step_coproc(&mut self, _main_mclk: u32, _scpu_mar: u32)

Step the cartridge coprocessor (SA-1 / Super FX / DSP-1 / …) forward by approximately main_mclk master cycles of main-CPU progress. Default = no-op for plain LoROM / HiROM carts.

Coproc mappers translate that to the right number of coproc instructions internally (e.g. SA-1 is 10.74 MHz ≈ 2× the main CPU’s max FastROM rate, so it consumes ~2 SA-1 instructions per main-CPU `mclk`). Implementations should be safely callable even when their coprocessor is reset / stopped.

scpu_mar is the S-CPU’s last bus-access address (ares cpu.r.mar), used by the SA-1 to model shared-bus conflict() contention; coprocs without bus contention ignore it.

fn coproc_main_irq_pending(&self) -> bool

true while the cartridge coprocessor is asserting an IRQ line onto the main CPU (SA-1 SCNT bit 7 latched + SIE bit 7 enabled, for instance). The main-CPU bus ORs this into its own irq_pending so the CPU services it through its normal IRQ path. Plain LoROM / HiROM carts return false.

fn coproc_ram(&self) -> Option<&[u8]>

Diagnostic: the coprocessor’s work RAM (e.g. Super FX Game Pak RAM) read directly, bypassing the SNES-side ownership gating that returns open-bus while the coprocessor owns the RAM. None if the mapper has no coprocessor RAM. Used to compare luna’s CPU-prepared GSU inputs against a reference.

fn sa1_snapshot(&self) -> Option<Sa1Snapshot>

Snapshot the SA-1 coprocessor’s CPU state, if this mapper hosts one. Plain LoROM / HiROM / Super FX / DSP-N return None. Used by luna-api to expose SA-1 PC / running state to debug integrations diagnosing main↔SA-1 mailbox deadlocks.

fn dsp1_snapshot(&self) -> Option<Dsp1Snapshot>

A read-only DSP-1 (uPD7725) state snapshot for the debugger, or None for non-DSP mappers.

fn enable_sa1_side_log(&mut self)

Enable SA-1-side execution logging: from now the coprocessor records its own accesses to the SA-1 MMIO window ($2200-$23FF) with its PC, for diagnosing why the SA-1 (re)asserts registers like SCNT. Complements the S-CPU-side --sa1-log. No-op for non-SA-1 mappers.

fn take_sa1_side_log(&mut self) -> Vec<Sa1SideEvent>

Drain the SA-1-side execution log (empty if disabled / not SA-1).

fn enable_sa1_trace(&mut self, _max_events: usize)

Enable a full SA-1 instruction trace: a pre-instruction register snapshot per SA-1 opcode (up to max_events), for diffing luna’s SA-1 PC/register stream against a reference emulator (bsnes/Mesen2) to localise the SMRPG deadlock. No-op for non-SA-1 mappers.

fn take_sa1_trace(&mut self) -> Vec<Sa1TraceEvent>

Drain the SA-1 instruction trace (empty if disabled / not SA-1).

fn enable_superfx_trace(&mut self, _max_events: usize)

Enable a full Super FX (GSU) instruction trace: a per-opcode snapshot of the GSU PC + register file (up to max_events), for diffing luna’s GSU stream against a reference (bsnes / siena) to localise rendering divergences. No-op for non-Super-FX mappers.

fn take_superfx_trace(&mut self) -> Vec<SuperFxTraceEvent>

Drain the Super FX instruction trace (empty if disabled / not GSU).
