1//! The [`SpcBus`] trait — the view the SPC700 CPU has of the system.
2//!
3//! The SPC700 has a flat 16-bit address space, with 64 KB ARAM
4//! occupying most of it. The "interesting" parts are:
5//!
6//! - `$F0-$FF` — DSP index/data, mailboxes, timers, control register.
7//!   Implementations route these to the matching subsystem state.
8//! - The rest is plain RAM.
9//!
10//! Unlike the 65C816 `Bus`, the SPC700 bus does **not** model cycle
11//! costs explicitly — the SPC700 is a fixed 1.024 MHz CPU with a
12//! simple per-opcode cycle table that we can apply post-execute.
13
14/// View of the system exposed to the SPC700.
15///
16/// # Per-cycle clocking (the ares/Mesen2 grammar)
17///
18/// On real hardware the SPC700 clocks the S-DSP **and** the three timers
19/// on **every** cycle — each memory access *and* each internal idle
20/// cycle — in position (ares `smp/memory.cpp`→`wait()`; Mesen2
21/// `Spc::IncCycleCount`). So a faithful consumer steps its timer/DSP
22/// state one cycle per [`read`](Self::read) / [`write`](Self::write) /
23/// [`idle`](Self::idle) call. Opcode implementations therefore call
24/// [`idle`](Self::idle) at each internal cycle (matching ares'
25/// `component/processor/spc700/instructions.cpp`), so the bus sees the
26/// exact hardware cycle sequence rather than just a post-hoc total.
27pub trait SpcBus {
28    /// Read one byte at the 16-bit address (one bus cycle).
29    fn read(&mut self, addr: u16) -> u8;
30
31    /// Write one byte at the 16-bit address (one bus cycle).
32    fn write(&mut self, addr: u16, value: u8);
33
34    /// An internal/idle cycle: no memory access, but the SPC still
35    /// burns a cycle and clocks the DSP + timers. Default is a no-op so
36    /// flat-RAM consumers (tests) that don't model timing need not care;
37    /// the timing-accurate consumer ([`luna_apu`]'s bus view) overrides
38    /// it to clock one cycle.
39    fn idle(&mut self) {}
40}
